Professor Layton 4 - Digit Puzzle

  • In this notebook, we will apply our search algorithm to find a solution for the 4 Digit puzzle from the game Prof. Layton and Curious Village
  • A, B, C and D are single-digit numbers. The 
following equations can all be made with 
these numbers.

A + C = D
A x B = C
C - B = B
A x 4 = D

Find the values of each digit and input 
your answer as a four digit-number, ABCD.

Solving the 4 Digit Puzzle

Brute force search would be quite slow for humans. Total number of possibilities would be 10 * 9 * 8 * 7 (if numbers are unique) or 10000 (if numbers can be repeated)

Humans would try and solve the equations

C = 3A

C = 2B

C must be divisble by 2 and 3, A = 2, B=3, C=6

NO_SOLUTION, SOLUTION, POSSIBLE_SOLUTION = 0,1,2

def is_valid( equation ):
  # A is at position 0, B is at position 1, ...

  if ( equation[0] is not None ) and ( equation[2] is not None ) and ( equation[3] is not None ) and ( equation[0] + equation[2] != equation[3] ):
    return NO_SOLUTION
 
  if ( equation[0] is not None ) and ( equation[1] is not None ) and ( equation[2] is not None ) and ( equation[0] * equation[1] != equation[2] ):
    return NO_SOLUTION

  if ( equation[2] is not None ) and ( equation[1] is not None ) and ( equation[2] - equation[1] != equation[1] ):
    return NO_SOLUTION

  if ( equation[0] is not None ) and ( equation[3] is not None ) and ( equation[0] * 4 != equation[3] ):
    return NO_SOLUTION
 
  if None in equation:
    return POSSIBLE_SOLUTION
  else:
    return SOLUTION

def find_digits( n_digits, dfs = True ):
  initial = [ None ] * n_digits
  queue = []
  g = gv.Digraph( comment="Prof. Layton's 4 Digits Puzzle" )
  img = 0

  queue.append( (initial,0, [] ) )
  num_nodes = len(queue)
  sols = []
  while( len(queue) > 0 ):
    current, level, parent = queue.pop()
    valid = is_valid(current)
    if valid == SOLUTION:
      sols.append( current)
      g.node( str(current), shape='box')
      if parent is not None and len(parent) > 0:
        g.edge( str(parent[-1]), str(current) )
        g.render( f"{dir}/g1_{img:05d}",format='png' )
        img = img + 1
      break
    elif valid == POSSIBLE_SOLUTION: 
      g.node( str(current) )   
      if parent is not None and len(parent) > 0:
        g.edge( str(parent[-1]), str(current) )
        g.render( f"{dir}/g1_{img:05d}",format='png' )
        img = img + 1
      #print("   " * level, current)
      for bi in range(len(current)): #-1,-1,-1):
        if current[bi] is None:
          for nb in range(9, -1,-1):
            if nb not in current:
              next = current.copy()
              next[bi] = nb
              if dfs:
                queue.append( (next, level+1, parent.copy() + [ current ] ) ) 
              else:
                queue.insert(0, (next, level+1, parent.copy() + [ current ] ) )             
              num_nodes = num_nodes + 1
          break
  return num_nodes, sols, g

Local Search

  • Depth first search and breadth first search are the two global search methods
    • They search the entire search space in a systematic way and can find the globally best solution
    • This can be very slow for large search spaces

Local Search

  • Hill Climbing: From the current state, we look at all possible moves and we pick the best move
    • Does not search the entire space any more (no global search)
    • Does find the best solution in the local neighborhood (local search)

4 Digits and Hill Climbing

  • Assume that we have some numbers A=1, B=5, C=3, D=3 and I try them as a solution
  • Then most likely, they will not work. We can test that
    • A+C=D => 1 + 3 = 3 Wrong!, Difference is +1
    • A*B=C => 1 * 5 = 3 Wrong!, Difference is +2
    • C-B=B => 3 - 5 = 5 Wrong!, Difference is -7
    • A*4=D => 1 * 4 = 3 Wrong!, Difference is +1
  • Can define error: 1 + 2 -7 + 1 = -3 - In this case, the error may be 0, but not all equations correct.
  • We must use either the absolute of the error abs(1) + abs(2) + abs(-7) + abs(1) = 11
  • Most commonly we would use the least sqaured error (LSE) as a error function: 1**2+2**2+(-7)**2+1**2 = 55

def d4_error_lse( equation ):
  # A is at position 0, B is at position 1, ...
  cum_err = 0

  if ( equation[0] is not None ) and ( equation[2] is not None ) and ( equation[3] is not None ):
    e1 = (equation[0] + equation[2] - equation[3])
    #print('e1',e1,e1**2)
    cum_err = cum_err + e1**2
    
  if ( equation[0] is not None ) and ( equation[1] is not None ) and ( equation[2] is not None ):
    e2 = ( equation[0] * equation[1] - equation[2] )
    #print('e2',e2,e2**2)
    cum_err = cum_err + e2**2

  if ( equation[2] is not None ) and ( equation[1] is not None ):
    e3 = ( equation[2] - equation[1] - equation[1] )
    #print('e3',e3,e3**2)
    cum_err = cum_err + e3**2

  if ( equation[0] is not None ) and ( equation[3] is not None ):
    e4 = ( equation[0] * 4 - equation[3] )
    #print('e4',e4,e4**2)
    cum_err = cum_err + e4**2
  return cum_err

def find_digits_hc( n_digits, initial = None ):
  if initial is None:
    initial = [ 5 ] * n_digits
  queue = []
  g = gv.Digraph( comment='4 Digits Puzzle - Hill Climbing' )
  img = 0

  current_node = ( initial,0, [] )
  sols = []
  num_nodes = 1

  while( current_node is not None ):
    current, level, parent = current_node
    err = d4_error_lse( current )
    print("  " * level, 'Current', current, 'Error', err )
    if err == 0.0:
      sols.append( current )
      g.node( str(current), shape='box' )
      if parent is not None and len(parent) > 0:
        g.edge( str(parent[-1]), str(current) )
        g.render( f"{dir}/ghc_{img:05d}",format='png' )
        img = img + 1
      break
    else:
      g.node( str(current) )   
      if parent is not None and len(parent) > 0:
        g.edge( str(parent[-1]), str(current) )
        g.render( f"{dir}/ghc_{img:05d}",format='png' )
        img = img + 1

      current_node = None
      min_error = err
      for pos in range(len(current)):
        digit = current[pos]
        for try_d in [ digit + 1, digit - 1 ]:
          if (try_d >= 1) and (try_d <= 9):
            cand = current.copy()
            cand[pos] = try_d
            try_err = d4_error_lse( cand )
            print("  " * level, 'Trying', cand, 'Error', try_err )
    
            if try_err < min_error:
              current_node = ( cand.copy(), level + 1, parent.copy() + [current] )
              num_nodes = num_nodes + 1
              min_error = try_err
  return num_nodes, sols, g

Local Search and Failure

  • Sometimes the local search will fail to find a solution that is good enough
  • The path of search through the search space is defined by the initial position
  • The (local) solution that we find is dependent on the start state
  • Simplest, but surprisingly effective, solution is to start with a different random initial position: Random restart

Local Search

  • The error function above breaks the space of possible solutions up into five areas
  • Success of hill climbing and other local methods depends on two factors
      • Reasonable chance of finding an initial point which will lead to the global maximum
      • Reasonable slope that we can follow to the global maximum

Local Search and Gradient Descent

  • Gradient descent is another local search method similar to hill climbing
  • Used for numerical approximation problems like regression
  • Gradient descent is at the heart of many machine learning (artificial neural networks (ANN) and deep learning (DL)) and optimization algorithms
  • We must use local methods because searching the entire space is too time consuming. Astronomically large search spaces
